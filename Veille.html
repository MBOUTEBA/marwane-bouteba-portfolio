Exemple 1 – Cybersécurité

Ransomware LockBit 4.0

Contexte observé

En novembre 2025, le ransomware LockBit 4.0 a refait surface. Il se propage via des e-mails frauduleux imitant des factures de fournisseurs habituels. Une fois le fichier joint ouvert, il chiffre les données et demande une rançon (vu sur ZATAZ et alerte Google « ransomware PME »).

Ce que j'ai retenu :

  • Le phishing reste la porte d'entrée n°1 (82 % des attaques selon les rapports).
  • Le MFA (authentification multifacteur : mot de passe + code sur téléphone ou appli) bloque 99 % des comptes compromis.
  • Le chiffrement des données au repos (via BitLocker sur Windows ou LUKS sur Linux) rend les données illisibles même si volées.

Application pratique

Ces bonnes pratiques peuvent être appliquées dans n'importe quelle infrastructure d'entreprise pour renforcer significativement la sécurité face aux ransomwares et tentatives de phishing.

Exemple 2 – Virtualisation & Cloud

Proxmox 8.3 - Backup incrémental

Contexte observé

Proxmox 8.3 est sorti avec une amélioration majeure : le backup incrémental natif sur ZFS. Au lieu de recopier tout un backup à chaque fois, il ne sauvegarde que les modifications (découvert sur IT-Connect et confirmé par des retours sur Reddit r/sysadmin).

Ce que j'ai retenu :

  • Un backup complet de 500 Go prenait 45 min avec Veeam.
  • Avec Proxmox 8.3 + ZFS incrémental : 8 min seulement pour le même volume.
  • Proxmox est open-source → zéro coût de licence, contrairement à VMware ou Veeam.

Potentiel d'application

Proxmox représente une alternative open-source performante qui pourrait réduire significativement les coûts de licence tout en améliorant les performances de backup.
