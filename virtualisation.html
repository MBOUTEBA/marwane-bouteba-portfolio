Loadz – Projet Virtualisation

Transformer un seul PC en trois serveurs sécurisés pour toute l'équipe

Un PC = Trois Serveurs

Serveur de fichiers, serveur web et base de données avec sauvegarde automatique

Objectif & Résultat

L'objectif de ce projet était de faire plus avec moins de matériel, en optimisant les ressources existantes.

Le résultat obtenu permet de faire fonctionner trois services différents sur une seule machine physique, pour huit utilisateurs, sans aucun achat supplémentaire, avec la possibilité de restaurer le système en seulement trente secondes.

Architecture du Projet

VMware Workstation Pro 17 sur PC Windows 11 Pro (16 Go RAM)

VM Rôle OS RAM CPU Disque
VM1 Fichiers Windows Server 2022 4 Go 2 40 Go
VM2 Web Debian 12 2 Go 1 25 Go
VM3 Base de données + Sauvegarde Debian 12 2 Go 1 30 Go

Réseau : VMnet2 – Host-Only (réseau interne isolé)

Machine IP Nom d'hôte
Mon PC 192.168.56.1 host
VM1 192.168.56.101 files.loadz.local
VM2 192.168.56.102 web.loadz.local
VM3 192.168.56.103 db.loadz.local

Étapes de Réalisation

1

Installation & Réseau

Téléchargement et installation de VMware Workstation Pro 17, création d'un réseau VMnet2 (Host-Only sans DHCP) et activation de la virtualisation dans le BIOS de la machine hôte.

2

Création des Machines Virtuelles

Chaque VM a été créée via le menu New Virtual Machine avec les ISO officiels de Windows Server et Debian. Attribution de RAM, CPU et disque selon le tableau d'architecture, puis connexion au réseau interne VMnet2.

3

Configuration IP & DNS

VM1 – Windows Server :

New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.56.101 -PrefixLength 24

VM2 & VM3 – Debian :

nano /etc/network/interfaces auto enp0s3 iface enp0s3 inet static address 192.168.56.102 # .103 pour VM3 netmask 255.255.255.0 systemctl restart networking

DNS local sur l'hôte (C:\Windows\System32\drivers\etc\hosts) :

192.168.56.101 files.loadz.local 192.168.56.102 web.loadz.local 192.168.56.103 db.loadz.local
4

Comptes Utilisateurs

Création d'un compte par membre de l'équipe pour isoler les accès. Configuration sur VM1 (Windows Server) :

New-LocalUser -Name "adelprat" -Password (ConvertTo-SecureString "Alex2025!" -AsPlainText -Force) Add-LocalGroupMember -Group "Users" -Member "adelprat","rg",... New-Item -Path "C:\Shares\Users\adelprat" -ItemType Directory icacls "C:\Shares\Users\adelprat" /grant "adelprat:(F)"
5

Serveur de Fichiers (VM1)

Configuration du partage réseau pour que chaque utilisateur dispose d'un espace personnel sécurisé :

New-SmbShare -Name "Users" -Path "C:\Shares\Users" -FullAccess "Administrators" -ChangeAccess "adelprat","rg",... Set-NetFirewallRule -DisplayGroup "File and Printer Sharing" -Enabled True net use Z: \\files.loadz.local\Users\g1 /user:g1 Graph1!
6

Serveur Web (VM2)

Installation et configuration du serveur Apache pour héberger le site du projet :

apt update && apt install apache2 -y systemctl enable apache2 ufw allow from 192.168.56.0/24 to any port 80 ufw enable
7

Base de Données et Sauvegarde (VM3)

Installation de MariaDB et configuration des accès :

apt install mariadb-server -y mysql_secure_installation mysql -e "CREATE DATABASE clients;" mysql -e "GRANT ALL ON *.* TO 'adelprat'@'%' IDENTIFIED BY 'Alex2025!';"

Script de sauvegarde automatique (quotidien à 2h du matin) :

nano /usr/local/bin/backup.sh #!/bin/bash DATE=$(date +%F) mkdir -p /backup/$DATE mysqldump -u root -p'MonMDPfort123!' --all-databases > /backup/$DATE/backup.sql chmod +x /usr/local/bin/backup.sh (crontab -l; echo "0 2 * * * /usr/local/bin/backup.sh") | crontab -
8

Snapshots

Des snapshots ont été réalisés à chaque étape importante : après l'installation, la configuration du réseau et la mise en place des services.

Cela permet de revenir instantanément à un état stable en cas d'erreur. Une restauration complète se fait en moins de 30 secondes.

Sécurité Appliquée

Réseau Isolé

Environnement réseau isolé sans accès Internet pour éviter toute intrusion extérieure.

Comptes Individuels

Chaque utilisateur dispose de son propre compte avec un mot de passe robuste pour la confidentialité des données.

Pare-feu Configuré

Un pare-feu est activé sur chaque VM pour limiter les connexions aux seuls ports nécessaires.

Sauvegardes Automatiques

Sauvegardes quotidiennes de la base de données et des fichiers partagés pour assurer la continuité du service.

Snapshots Systématiques

Snapshots avant chaque modification importante pour une traçabilité complète et restauration rapide.

Conclusion

Un projet réussi

Ce projet m'a permis de démontrer qu'il est possible de consolider plusieurs services essentiels (fichiers, web et base de données) sur un seul poste physique, tout en maintenant un haut niveau de sécurité et de fiabilité.

Grâce à la virtualisation, j'ai pu créer un environnement professionnel stable, isolé et performant, permettant à l'équipe Loadz de travailler efficacement sans investissement matériel supplémentaire.

Les sauvegardes, la gestion des utilisateurs et les snapshots assurent la sécurité et la pérennité des données.

En résumé : ce projet prouve qu'un seul ordinateur bien configuré peut remplir le rôle de trois serveurs distincts, offrant un excellent équilibre entre économie, performance et sécurité.
